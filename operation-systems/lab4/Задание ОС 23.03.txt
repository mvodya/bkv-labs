Дистанционная лабораторная №1.

"Режимы обслуживания очередей краткосрочного планирования".
Внимание! Для удаленной работы с графическим приложением под
Windows установите X-сервер (как войти в компьютеры удаленно
написано далее).

Приложение VisualOS позволяет наблюдать за работой планировщика
с использованием алгоритмов обслуживания FCFS, SPN, SRT, RR, HRRN.
Для исследования подготовьте сценрий запуска (файл), состоящий из множества
запланированных процессов, отсортированных по времени запуска. Описание
одного процесса в файле: 

proc {
	start_time=6  // время запуска
	end_time=16   // продолжительность процесса от 5 до 40 случайно
                     // кроме 2х-3х сверхдлинных процессов
	IO{
          //  пустая секция
	}
	MEM{
		page=5 write=0
            //   одна страница с номером от 1 до 10 случайно
	}
}
 Таких процессов нужно записать до 5000 штук, причем среди них - два-три
"долгоживущих процесса", а остальные - короткие, но создающие постоянную очередь,
в режимах SRT и SPN. Длина очереди не должна превышать 10-20 процессов. Если
образуется больше процессов, - завершайте приложение скорее! Или "повесите"
удаленный компьютер. Скорректируйте тогда параметры продолжительности или
время запуска. Для проверки другого алгоритма, перезапустите VisualOS и
загрузите сценарий заново.
  Задача: создать сценарий (конечно программным способом, с
использованием генератора псевдослучайных чисел), показывающий наглядно при
запуске наличия голодания для тех алгоритмов, где они возможны.
Отчет: файл сценария или программа, его генерирующая.
Перез запуском VisualOS из командной строки измените переменную окружения
LANG: 
export LANG=en_US
иначе русские слова в верхнем меню невозможно будет прочесть.

Сейчас сетевые администраторы настроили заход в класс дистанционно!

Сейчас можно во время проведения пар по расписанию работать удаленно на компьютерах
в классе 1521. Режим работы - по протоколу SSH. Из Windows - запустите приложение
PuTTY, выберите соединение по ssh, адрес доступного компьютера - 217.74.117.18,
вход с Вашей учетной записью. Вы попадете на компьютер 10.40.14.101, далее можно
перейти на тот компьютер, на котором Вы работали постоянно командой ssh -Y 10.40.14.1xx,
где xx- номер компьютера в классе (от 01 до 14). Нумерация по часовой стрелке от шкафа
(01-й) до двери (14-й). Вводите свой пароль при входе. 
Консольный файловый менеджер - mc. Для скрытия панелей и их открытия используйте Ctrl/o.
Выход из сеанса - exit. Если был запущен mc - то еще раз exit.
Редактор - mcedit или vim. Для запуска графических приложений удаленно помимо PuTTY
в Windows нужно установить X-сервер, можно взять на 5.100.98.127/ftp/Tools/
vcxsrv-64.1.20.6.0.installer.exe . Там же в каталоге "ftp/Дистанционная работа" можно
скачать PuTTY.
В сеансе PuTTY указать проброс X11-протокола. Тогда можно работать с geany, запуская его
в командной строке.
С компьютера с установленной ОС Linux: 
ssh -Y login@217.74.117.18 (вместо login вводим свой логин, далее пароль на вход).
ssh -Y 10.40.14.1xx (вместо хх - номер компьютера в классе). Можно работать в geany,
запустив его из командной строки. Завершение сеанса - закрыть приложения и exit.

Если Вы работаете на домашнем компьютере, поместите в конце пары (или позже) наработанный за 
это время код в домашний каталог на компьютер 217.74 117.18 или любой другой в классе.
